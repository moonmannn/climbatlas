# ClimbAtlas Editorial Copy

Write clear, decision-useful ClimbAtlas copy while preserving factual uncertainty and bilingual meaning.

## Workflow

1. Identify or infer the content mode: UI, homepage, destination, route card, route detail, DNA quiz, DNA archetype, DNA Match, or localization. Ask only when the wrong mode would materially change the result.
2. Read `../../../docs/content/brand-voice.md` and the relevant section of `../../../docs/content/content-patterns.md`.
3. Read `../../../docs/content/banned-patterns.md` before revising existing copy. Use `../../../docs/content/approved-examples.md` when calibrating tone or bilingual output.
4. Gather the available facts, sources, editorial fields, inferred fields, character limits, and neighboring copy. Preserve missing information as unavailable.
5. Separate sourced fact, ClimbAtlas editorial judgment, and inferred DNA attributes. Do not make one category sound like another.
6. Draft for user decisions first. Add atmosphere only after the copy answers the relevant fit or trade-off question.
7. For bilingual work, establish one meaning brief, then write natural English and natural Chinese with equal certainty and personality. Do not translate syntax literally.
8. Self-review for unsupported claims, internal labels, generic marketing language, repeated templates, metaphor density, ambiguity, and encoding defects.

## Output

- Match the repository field structure when editing code or data.
- When drafting outside code, label the content mode and provide English and Chinese as paired blocks when both are requested.
- Add a short factual-verification note when the draft contains facts that were not verified during the task.
- Keep UI copy plain, route cards compact, destination copy balanced, and DNA archetypes the most expressive mode.
- Do not rewrite unrelated website copy or expand the requested scope automatically.

## Non-negotiable boundaries

- Do not invent route facts, history, movement, rock or protection quality, safety, access, conditions, popularity, beta, or user opinion.
- Do not expose internal workflow labels on public pages.
- Do not describe a DNA match as ability, readiness, safety, or certainty.
- Do not copy external route descriptions, beta, guidebook text, reviews, ratings, or unlicensed media.
- Do not repair corrupted source text by guessing when the intended original cannot be established.
